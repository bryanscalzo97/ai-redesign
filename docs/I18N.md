# Internationalization (i18n)

The app ships with **English**, **Spanish**, **French**, and **Portuguese**. Language detection, switching, and persistence are handled automatically via `react-i18next`.

## How it works

1. `src/i18n/index.ts` initializes i18next
2. On launch, language is resolved: **stored preference → device locale → fallback**
3. Translations are in `src/i18n/locales/` (one file per language)
4. Components use `useTranslation()` and call `t("key")`

## Configuration

Language settings live in `src/config/app.ts`:

```ts
i18n: {
  enabled: true,
  defaultLanguage: "device",
  fallbackLanguage: "en",
  supportedLanguages: ["en", "es", "fr", "pt"],
}
```

| Field                | Type                  | Description                                    |
| -------------------- | --------------------- | ---------------------------------------------- |
| `enabled`            | `boolean`             | Toggles multi-language support                 |
| `defaultLanguage`    | `"device"` / `string` | `"device"` auto-detects, or force a language   |
| `fallbackLanguage`   | `string`              | Used when device language isn't supported       |
| `supportedLanguages` | `string[]`            | Must match locale files in `src/i18n/locales/` |

## Adding a new language

1. **Create the locale file** — Copy `src/i18n/locales/en.ts` to e.g. `de.ts` and translate all strings

2. **Register the resource** — Import and add it in `src/i18n/index.ts`

3. **Update appConfig** — Add the language code to `supportedLanguages` in `src/config/app.ts`

## Using translations

```tsx
const { t } = useTranslation();
t("common.continue");     // "Continue" / "Continuar" / etc.
t("camera.takePhoto");    // namespace:key format
```

## File Map

```
src/config/app.ts          ← i18n config
src/i18n/index.ts          ← i18next initialization
src/i18n/locales/en.ts     ← English (source of truth)
src/i18n/locales/es.ts     ← Spanish
src/i18n/locales/fr.ts     ← French
src/i18n/locales/pt.ts     ← Portuguese
```
